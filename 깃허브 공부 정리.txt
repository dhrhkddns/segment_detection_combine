pull = fetch + merge

push 로컬 저장소 -> 원격 저장소 (컴퓨터에서 깃허브) 정보를 밀어 넣는다.

pull 원격 저장소 -> 로컬 저장소 (깃허브에서 내 컴퓨터로) 정보를 가져온다.

fetch 읽는다? 안전하게? 조금더 알아보자.

merge 병합한다. 내 집 컴퓨터, 피시방 컴퓨터가 있을때 서로 다른 두곳에서 작업을 했을때

서로 둘다 다른 커밋을 하고 push를 하고 pull하면 덮어쓰기로 지워지기 때문에

깃허브는 이를 방지하기 위해 에러를 낸다. 고로. 이런경우에는

두개의 수정 파일을 갖기 위해 fetch와 merge로 branch를 통합한다.

(더 알아볼것 필요하다면)

commit 커밋 작업때 수정한 사항들을 원격에 올릴때 최종 후보들을 

진짜 올릴것들을 선별하는 과정. push 할때는 commit 한것들만 push가 된다.

clone 아예 통째로 복사한다.
